Historia da web (voltar)

A historia da web ou internet tem inicio la nos anos 1960 com a visão pioneira de Joseph Carl sobre uma "rede galáctica". A teoria de uma rede galáctica de Joseph Carl tratava se de uma rede de comunicação global, onde computadores pudessem trocar informações entre si. Porem a tecnologia naquela época, não estava apta para isso. Foi então que entrou em cena o grupo de cientistas da ARPANET. Eles eram uma turma de cientistas que trabalhava para o governo dos EUA, e eles começaram a construi os alicerces do que viria a ser a internet.

ARPANET era uma rede de computadores que permitia que diferentes máquinas conversassem entre si permitindo assim permitindo que a primeira palavra fosse comunicada entre dois pontos diferentes, essa palavra foi "Login" porem a maquina receptora recebeu apenas o LO . Isso aconteceu lá na década de 1970, e foi tipo o embrião da internet Eles estavam fazendo isso principalmente para fins militares e acadêmicos o que restringia o acesso a tal tecnologia, mas a semente estava plantada.

Então, nos anos 80, um cara chamado Tim Berners-Lee apareceu. Ele tinha uma ideia genial: em vez de apenas conectar computadores, por que não criar uma maneira de conectar informações? Ele desenvolveu a ideia de hipertexto , que é como um texto onde você pode clicar em palavras ou links para ir para outras informações relacionadas. Isso era o que ele chamava de "World Wide Web" ou simplesmente "web".

Apartir disso ele construiu o primeiro navegador e o primeiro servidor web. O navegador era como a sua ferramenta para explorar essa nova web, e o servidor era onde as informações eram armazenadas e compartilhadas. E assim, em 1991, a primeira página da web foi ao ar. Era uma página simples, só explicava como tudo funcionava, mas foi um começo emocionante!

Daí pra frente, as coisas começaram a explodir. Novos navegadores foram criados, como o Mosaic e o Netscape Navigator. As pessoas estavam começando a criar seus próprios sites, compartilhar informações e, de repente, a web estava se tornando um lugar para conexões globais.

Comunicação client x server (voltar)

A comunicação cliente-servidor é um modelo, que descreve a interação entre dois componentes principais: o cliente e o servidor. Esses componentes podem ser programas de software, dispositivos físicos ou até mesmo sistemas distribuídos. A comunicação entre eles ocorre por meio de troca de mensagens, onde o cliente envia solicitações ao servidor, e o servidor responde a essas solicitações com os resultados ou informações solicitadas.

A comunicação entre o cliente e o servidor é regulada por protocolos. Um protocolo define as regras e a estrutura da comunicação, especificando como os dados devem ser formatados e transmitidos. Alguns exemplos de protocolos são HTTP/HTTPS (utilizados na web) nos quais o primeiro nao tem criptografia dos pacotes e o segundo tem.

O que sao Clients? (voltar)

Um "client" (cliente, em português) é aquele que solicita serviços, recursos ou informações de um servidor. Os clientes desempenham um papel ativo na interação, iniciando a comunicação ao enviar solicitações para um servidor e aguardando as respostas correspondentes

O que sao Servers? (voltar)

Um "server" (servidor, em português) é um componente ou programa de software que fornece serviços, recursos ou informações a clientes. Em um modelo de comunicação "Client X Server" (Cliente X Servidor, em português), os servidores desempenham um papel passivo, esperando por solicitações de clientes e respondendo a essas solicitações com os resultados ou dados solicitados./p>

O que é Html? (voltar)

HTML (Hypertext Markup Language) é uma linguagem de marcação de Hipertexto, e é uma linguagem essencial para a criação de páginas da web. Ela serve como o alicerce sobre o qual os conteúdos da internet são construídos. Utilizando uma série de "tags" para estruturar o conteúdo, como texto, imagens e links, permitindo que os navegadores exibam essas informações de maneira formatada.

O HTML permite aos desenvolvedores definir a estrutura lógica de uma página, como cabeçalhos, parágrafos, listas, links, imagens e outros elementos, organizando o conteúdo de forma hierárquica. Os elementos HTML são compostos por tags, que são códigos inseridos entre colchetes angulares ("<" e ">"). Cada tag descreve um elemento específico da página, como um título, um parágrafo, uma imagem, um link, etc. As tags podem conter atributos que fornecem informações adicionais sobre o elemento, como seu estilo, comportamento ou características específicas.

Estrutura basica do Html (voltar)

A estrutura básica do HTML é composta por três partes fundamentais. A primeira parte é a declaração de que o documento é HTML, realizada através da tag < html >. A segunda parte é definida pela tag < head >, e a terceira parte é representada pela tag < body >. As tags < head > e < body >, que compõem respectivamente o cabeçário e o corpo, devem ser declaradas dentro da tag < html > .

CABEÇÃRIO*

Declarado pela tag < head >, o cabeçário é a parte do código que contém informações essenciais sobre o documento, como o título da página, O cabeçãrio e parte do codigo que sera carregada primeiro antes de ser exibido para o usuario

CORPO*

Declarado pela tag < body >, o corpo é a parte do código que contém todo o conteúdo visível da página. Isso inclui textos, imagens, vídeos, links e outros elementos que são apresentados aos visitantes do site.

Relação Tag X Atributo (voltar)

No HTML, as tags são usadas para definir diferentes elementos e estruturas em uma página da web, enquanto os atributos são usados para fornecer informações adicionais sobre esses elementos. Os atributos são inseridos dentro das tags e geralmente têm um nome e um valor.Como nos exemplos abaixo:

  1. Tag < a > (Âncora):
    • Atributo "href": Define o URL de destino do link.
    • Atributo "target": Especifica onde abrir o link (por exemplo, em uma nova janela ou na mesma janela).
  2. Tag < img > (Imagem):
    • Atributo "src": Indica a URL da imagem.
    • Atributo "alt": Fornece uma descrição alternativa da imagem (para acessibilidade ou quando a imagem não é exibida).
  3. Tag < h1 > (Título de nível 1):
    • Não tem atributos específicos, mas pode ser estilizado usando CSS.
  4. Tag < p > (Parágrafo):
    • Não tem atributos específicos, mas pode ser estilizado usando CSS.
